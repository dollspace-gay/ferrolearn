## Summary

`LassoCV` mirrors scikit-learn's `sklearn.linear_model.LassoCV` (`_coordinate_descent.py:1875`,
`class LassoCV(RegressorMixin, LinearModelCV)`): it evaluates a grid of `alpha`
regularization values with k-fold cross-validation, selects the alpha that minimizes the
mean CV mean-squared-error, and refits a `Lasso` on the full data with that alpha. When no
explicit alpha grid is supplied, the grid is auto-generated by the `_alpha_grid` formula
(`_coordinate_descent.py:96`). The ferrolearn implementation lives in
`ferrolearn-linear/src/lasso_cv.rs` (`LassoCV<F>`, `FittedLassoCV<F>`).

## Requirements

- REQ-1: Auto alpha-path generation matching `_alpha_grid`: `alpha_max = max(|Xᵀy|)/n_samples`
  on **centered** X,y (for l1_ratio=1 single-target), a log-spaced grid of `n_alphas` from
  `alpha_max` down to `alpha_max * eps` with `eps = 1e-3` and `n_alphas = 100` defaults.
- REQ-2: k-fold CV alpha selection — for each candidate alpha, fit `Lasso` on each train fold,
  score MSE on the held-out fold, pick the alpha minimizing mean CV MSE, then refit on full
  data; expose `alpha_` (best alpha), `coef_`, `intercept_`.
- REQ-3: Explicit user-supplied alpha grid (`alphas=`) bypassing auto-generation, with
  non-negativity / non-empty validation.
- REQ-4: `predict` (`X @ coef_ + intercept_`), `fit_intercept` handling, and
  `HasCoefficients` introspection (`coef_`, `intercept_`).
- REQ-5: CV fold strategy matches sklearn `check_cv(5) → KFold(5)` non-shuffled **contiguous**
  block splits, so the selected `alpha_` matches sklearn.
- REQ-6: `mse_path_` fitted attribute — the per-alpha, per-fold MSE grid retained on the
  fitted model.
- REQ-7: `alphas_`, `dual_gap_`, `n_iter_` fitted attributes carried from the final refit.
- REQ-8: `eps` exposed as a constructor parameter (sklearn `LassoCV(eps=1e-3)`), not hardcoded.
- REQ-9: `positive` constructor parameter (non-negative-coefficient constraint).
- REQ-10: `precompute` and `n_jobs` constructor parameters (Gram precompute; parallel folds).
- REQ-11: `random_state` and `selection` (`"cyclic"`/`"random"`) constructor parameters.
- REQ-12: ferray substrate — array/linalg on `ferray-core`/`ferray::linalg`, not `ndarray`/`faer`.

## Acceptance criteria

- AC-1: For `X = [[1]..[20]]`, `y = 2x+1`, `n_alphas=10`, `eps=1e-3`, the auto grid first value
  equals sklearn `_alpha_grid(...)[0] = 66.5` and last equals `0.0665` (live oracle).
- AC-2: `LassoCV(n_alphas=10, cv=3).fit` on the same data yields `coef_`, `intercept_`, and
  `alpha_` matching `sklearn.linear_model.LassoCV(n_alphas=10, cv=3)` within a documented
  tolerance, given identical fold assignment.
- AC-3: With `alphas=[0.001,0.01,0.1,1.0,10.0]`, the selected `best_alpha` is one of the
  supplied candidates.
- AC-4: `predict` returns `X @ coef_ + intercept_`; `coefficients()`/`intercept()` expose the
  refit Lasso's parameters.
- AC-5: sklearn `KFold(5).split(arange(20))` fold-0 test indices are `[0,1,2,3]` (contiguous);
  the ferrolearn fold-0 test indices must match (currently `[0,5,10,15]`).
- AC-6: `mse_path_` shape and values match `sklearn LassoCV(...).mse_path_`.

## Architecture

`LassoCV<F>` (`lasso_cv.rs`) is the unfitted estimator with fields
`alphas: Option<Vec<F>>`, `n_alphas: usize`, `cv: usize`, `max_iter: usize`, `tol: F`,
`fit_intercept: bool` — a subset of sklearn's `LinearModelCV.__init__` signature
(`_coordinate_descent.py:1506-1536`); the remaining sklearn params (`eps`, `precompute`,
`copy_X`, `verbose`, `n_jobs`, `positive`, `random_state`, `selection`) are absent
(REQ-8/9/10/11). Builder methods (`with_alphas`, `with_n_alphas`, `with_cv`, `with_max_iter`,
`with_tol`, `with_fit_intercept`) mirror the constructor kwargs; defaults are `alphas=None`,
`n_alphas=100`, `cv=5`, `max_iter=1000`, `tol=1e-4`, `fit_intercept=true`, matching sklearn
(`_coordinate_descent.py:1508-1514`).

`Fit for LassoCV` (`fn fit`) validates shapes (`ShapeMismatch`), `cv >= 2`
(`InvalidParameter`), and `n_samples >= cv` (`InsufficientSamples`), then:
1. Builds the alpha grid — explicit (validated) or auto via `compute_alpha_max` + `logspace`
   (REQ-1/REQ-3). Degenerate `alpha_max <= 0` falls back to `vec![1e-6; n_alphas]`, a coarser
   analog of sklearn's `np.finfo(float).resolution`-fill (`_coordinate_descent.py:180-183`).
2. Partitions indices with `kfold_indices` (round-robin `i % k`; **diverges** from sklearn's
   contiguous `KFold` — REQ-5/#421).
3. For each alpha, runs the folds via the `Lasso` estimator (`crate::Lasso`), accumulates
   `mse`, and keeps the lowest-mean-MSE alpha (REQ-2).
4. Refits `Lasso` with `best_alpha` on the full data and stores `best_alpha`, the full
   `alphas` grid, `coefficients`, and `intercept` in `FittedLassoCV<F>`.

`FittedLassoCV<F>` carries `best_alpha`, `alphas`, `coefficients`, `intercept`. It does NOT
carry `mse_path_` (REQ-6/#422) nor `dual_gap_`/`n_iter_` (REQ-7/#423). `Predict` and
`HasCoefficients` are implemented (REQ-4). Free helpers `mse`, `select_rows`,
`select_elements`, `compute_alpha_max`, `logspace`, `kfold_indices` are private.

Centering invariant: `compute_alpha_max` centers both X (per-feature mean) and y when
`fit_intercept`, matching sklearn's `_preprocess_data` centering before `Xy` is formed
(`_coordinate_descent.py:156-161`).
