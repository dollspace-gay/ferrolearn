## Summary
`ElasticNetCV<F>` mirrors scikit-learn's `sklearn.linear_model.ElasticNetCV` (`_coordinate_descent.py:2131`): for each candidate `l1_ratio` it builds a log-spaced alpha grid (`_alpha_grid`, `_coordinate_descent.py:96`), runs k-fold cross-validation fitting an `ElasticNet` per `(alpha, l1_ratio)` fold, selects the pair minimizing mean CV MSE, and refits on the full data. The numerical CV-selection mechanism is in place; the *selected* `(alpha, l1_ratio)` currently diverges from sklearn because the fold partition is round-robin (not contiguous KFold) and the default `l1_ratio` grid differs from sklearn's single `0.5`.

## Requirements
- REQ-1: Per-`l1_ratio` automatic alpha grid generation matching `_alpha_grid`, including the `/l1_ratio` factor: `alpha_max = max|Xᵀy_centered| / (n_samples * l1_ratio)`, then `n_alphas` geomspace points from `alpha_max` to `alpha_max * eps` (`eps=1e-3`).
- REQ-2: Joint `(alpha, l1_ratio)` k-fold CV selection — fit `ElasticNet` per fold, accumulate test-fold MSE, argmin mean MSE over the grid, then refit on the full training set with the winning pair.
- REQ-3: User-supplied `l1_ratios` grid is honored (builder `with_l1_ratios`).
- REQ-4: `predict` (`X @ coef + intercept`), `fit_intercept` threading, and `HasCoefficients` introspection (`coef_`/`intercept_` analogs).
- REQ-5: Cross-validation fold partition matches sklearn's non-shuffled `KFold` (contiguous blocks), so the selected `(alpha, l1_ratio)` is numerically reproducible against sklearn.
- REQ-6: Default `l1_ratio` matches sklearn's `ElasticNetCV()` default of a single `0.5`, so `ElasticNetCV::new()` reproduces `ElasticNetCV()` output by default.
- REQ-7: Fitted `mse_path_` attribute (shape `(n_l1_ratio, n_alpha, n_folds)`).
- REQ-8: Fitted `alphas_`, `l1_ratio_`, `dual_gap_`, `n_iter_` attributes.
- REQ-9: User-tunable `eps` constructor parameter (path length).
- REQ-10: `positive` constraint forcing non-negative coefficients.
- REQ-11: `n_jobs` parallel fold dispatch and `precompute` Gram-matrix caching.
- REQ-12: `random_state` / `selection` (`'cyclic'` vs `'random'`) coordinate-descent selection.
- REQ-13: ferray substrate (`ferray-core` / `ferray::linalg`) replacing the `ndarray`/`faer` substrate (R-SUBSTRATE-2).
- REQ-14: `l1_ratio == 0` handling matches sklearn — `_alpha_grid` raises `ValueError` ("Automatic alpha grid generation is not supported for l1_ratio=0"); ferrolearn must not silently auto-generate a grid for `l1_ratio=0`.

## Acceptance criteria
- AC-1: For `X = arange(1,21).reshape(20,1)`, `y = 2X+1`, `fit_intercept=True`, the generated grid's first element equals `max|Xᵀy_centered| / (n * l1_ratio)`: `133.0` for `l1_ratio=0.5`, `66.5` for `l1_ratio=1.0`; last element = first × `1e-3`.
- AC-2: For a fixed explicit `l1_ratios` and `alphas`, the selected `best_alpha`/`best_l1_ratio` is the grid pair minimizing mean CV MSE under the (current) fold partition.
- AC-3: `with_l1_ratios(vec![0.5, 0.9])` produces a 2-element search; an empty grid errors with `InvalidParameter`.
- AC-4: `predict` returns `len(X)` values; `coefficients().len() == n_features`; `with_fit_intercept(false)` yields a zero-bias decision function.
- AC-5 (REQ-5, gated): with contiguous KFold, `ElasticNetCV(cv=k)` `alpha_`/`l1_ratio_` match sklearn's `alpha_`/`l1_ratio_` on the diabetes dataset within sklearn's argmin determinism.
- AC-6 (REQ-6, gated): `ElasticNetCV::new()` searches the single `l1_ratio=0.5`, matching `ElasticNetCV()`.

## Architecture
The unfitted estimator is `ElasticNetCV<F>` (`elastic_net_cv.rs`) carrying `l1_ratios: Vec<F>`, `n_alphas`, `cv`, `max_iter`, `tol`, `fit_intercept`; builders (`with_l1_ratios`, `with_n_alphas`, `with_cv`, `with_max_iter`, `with_tol`, `with_fit_intercept`) and `Default`/`new`. It maps to sklearn's `class ElasticNetCV(RegressorMixin, LinearModelCV)` (`_coordinate_descent.py:2131`), whose CV machinery lives in `LinearModelCV.fit` (`_coordinate_descent.py:1552`).

`Fit for ElasticNetCV` is the translation of `LinearModelCV.fit`:
1. Validation gates (shape, non-empty `l1_ratios`, ratios in `[0,1]`, `cv >= 2`, `n_samples >= cv`, `n_alphas >= 1`) precede the search. sklearn validates via `_parameter_constraints` and `check_cv`.
2. `kfold_indices(n_samples, cv)` builds the fold partition. **Divergence (REQ-5/#431):** round-robin `i % k`, whereas sklearn's `check_cv(None) → KFold(5)` yields contiguous blocks (first `n%k` folds size `n//k + 1`, sequential).
3. Outer loop over `l1_ratios`: `compute_alpha_max_enet` (centered `max|Xᵀy|/(n·l1_ratio)`, mirroring `_alpha_grid` `:178`) feeds `logspace` (geomspace, `:185`). Inner loop over alphas × folds fits `ElasticNet` per fold and averages test-MSE.
4. `argmin` over the `(l1_ratio, alpha)` grid by mean CV MSE (`if avg_mse < best_mse`), mirroring `_coordinate_descent.py:1793-1799`.
5. Refit `ElasticNet` with `(best_alpha, best_l1_ratio)` on full data; store `coefficients`/`intercept` (`_coordinate_descent.py` Notes `:2274`).

The fitted type `FittedElasticNetCV<F>` holds `best_alpha` (`alpha_`), `best_l1_ratio` (`l1_ratio_`), `coefficients` (`coef_`), `intercept` (`intercept_`). `Predict` is `X @ coef + intercept` with a feature-count guard; `HasCoefficients` exposes `coef_`/`intercept_`. Not retained relative to sklearn: `mse_path_`, `alphas_`, `dual_gap_`, `n_iter_` (REQ-7/REQ-8).

Invariants: each `l1_ratio` gets its own alpha grid (the `/l1_ratio` factor scales `alpha_max` inversely with `l1_ratio`); centering of X and y is gated on `fit_intercept` consistently across grid generation and every per-fold fit. The substrate is `ndarray`/`faer` throughout (REQ-13/#439).
